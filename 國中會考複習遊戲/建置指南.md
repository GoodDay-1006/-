# 複習闖關王網站 — 完整建置指南

## 📁 檔案清單

| 檔案 | 用途 |
|------|------|
| `index.html` | 登入頁（輸入 Email 取得魔法連結） |
| `courses.html` | 課程選單頁（登入後才能看） |
| `bio1.html` | 生物第一冊遊戲（把你原本的檔案改名） |
| `geo6.html` | 地科第六冊遊戲（同上） |

---

## 🪜 完整步驟（照順序做）

---

## STEP 1：建立 Supabase 專案

**預計時間：10 分鐘**

1. 前往 https://supabase.com 點「Start your project」
2. 用 GitHub 帳號登入（建議）
3. 點「New project」
4. 填寫：
   - **Name**：review-game（或任意名稱）
   - **Database Password**：設一個強密碼並記下來
   - **Region**：選「Northeast Asia (Tokyo)」（台灣最近）
5. 點「Create new project」，等待 1–2 分鐘建立完成

---

## STEP 2：取得 Supabase 金鑰

1. 進入你的專案後，點左側選單「⚙️ Project Settings」
2. 點「API」
3. 找到以下兩個值並複製：

```
Project URL：
https://abcdefghijk.supabase.co
               ↑ 這串就是你的「專案ID」

anon public key（很長的一串 JWT）：
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiO...
```

4. 把這兩個值填入 `index.html` 和 `courses.html` 裡標有 ⚙️ 的地方：

```javascript
const SUPABASE_URL = 'https://abcdefghijk.supabase.co'  // 貼上 Project URL
const SUPABASE_ANON_KEY = 'eyJhbGciOi...'               // 貼上 anon key
```

---

## STEP 3：建立學員白名單資料表

1. 在 Supabase 專案頁，點左側「🗄️ Table Editor」
2. 點「New table」
3. 填寫：
   - **Name**：`allowed_emails`
   - 關閉「Enable Row Level Security (RLS)」（先關掉，方便測試）

> ⚠️ **安全提醒：正式上線前務必重新開啟 RLS，並在 Supabase「Table Editor → allowed_emails → RLS」新增 Policy，限制只有登入用戶才能查自己的資料。未開啟 RLS 時，任何人都能讀取全部學員 Email！**

4. 在「Columns」新增欄位：

| Column Name | Type | 說明 |
|-------------|------|------|
| `id` | int8 | 預設自動建立，保留 |
| `email` | text | 學員 Email，設為 unique |
| `name` | text | 學員姓名（選填） |
| `created_at` | timestamptz | 預設自動建立，保留 |

5. 點「Save」

---

## STEP 4：新增付費學員 Email

### 方法一：網頁介面（少量學員）
1. 點「🗄️ Table Editor」→「allowed_emails」
2. 點「Insert row」
3. 填入 Email 和姓名，點「Save」
4. 重複新增所有學員

### 方法二：CSV 批次匯入（大量學員）
1. 建立一個 `students.csv` 檔：
```csv
email,name
student1@gmail.com,王小明
student2@gmail.com,李小花
student3@hotmail.com,張大同
```
2. 點「🗄️ Table Editor」→「allowed_emails」
3. 點右上角「Import data」→ 上傳 CSV
4. 對應欄位後點「Import」

---

## STEP 5：設定 Magic Link 信件內容

1. 點左側「🔑 Authentication」→「Email Templates」
2. 點「Magic Link」
3. 修改信件內容（以下是建議模板）：

**Subject（主旨）：**
```
你的複習闖關王登入連結
```

**Body（內文，HTML 格式）：**
```html
<h2>你好！</h2>
<p>你剛才請求登入複習闖關王學員專區。</p>
<p>請點擊以下連結登入（15 分鐘內有效，只能使用一次）：</p>
<p><a href="{{ .ConfirmationURL }}" 
      style="background:#63d2be;color:#0d1a18;padding:12px 24px;
             border-radius:8px;text-decoration:none;font-weight:bold;">
  點我登入學員專區
</a></p>
<p style="color:#888;font-size:12px;">
  若你沒有請求此連結，請忽略此信。<br>
  此連結將在 15 分鐘後自動失效。
</p>
```

4. 點「Save」

---

## STEP 6：設定允許的網站網址

1. 點左側「🔑 Authentication」→「URL Configuration」
2. 在「Site URL」填入你的網站網址：
   - 如果還沒部署：填 `http://localhost:3000`（之後再改）
   - 如果已有網址：填 `https://你的網域.com`
3. 在「Redirect URLs」新增：
   ```
   https://你的網域.com/courses.html
   http://localhost:5500/courses.html
   ```
   （後者是本機測試用）

---

## STEP 7：整理你的遊戲檔案

把你之前製作的遊戲 HTML 改名放入資料夾：

```
magic-link-site/
├── index.html       ← 登入頁（已提供）
├── courses.html     ← 課程選單（已提供）
├── bio1.html        ← 把「生物闖關王_第一冊.html」改名為此
└── geo6.html        ← 把「地科闖關王_第六冊.html」改名為此
```

> ⚠️ 注意：遊戲 HTML 本身不需要修改，學員驗證在 courses.html 完成。

---

## STEP 8：部署到 Cloudflare Pages

**預計時間：5 分鐘**

### 方法一：直接上傳（最簡單）
1. 前往 https://pages.cloudflare.com，用 Cloudflare 帳號登入
2. 點「Create a project」→「Direct Upload」
3. 設定專案名稱，點「Create project」
4. 把整個遊戲資料夾拖進上傳區，點「Deploy site」
5. 完成！Cloudflare 會給你一個 `xxx.pages.dev` 網址

### 方法二：透過 GitHub（推薦，之後更新更方便）
1. 把遊戲資料夾上傳到 GitHub Repository
2. 在 Cloudflare Pages 連結此 Repository（「Connect to Git」）
3. Framework preset 選「None」，Build command 留空
4. 每次更新 GitHub，Cloudflare Pages 自動重新部署

---

## STEP 9：更新 Supabase 網址設定

部署完成後，把 Cloudflare Pages 給的網址填回 Supabase：

1. 「🔑 Authentication」→「URL Configuration」
2. 把「Site URL」改成 Cloudflare Pages 網址：`https://xxx.pages.dev`
3. 「Redirect URLs」新增：`https://xxx.pages.dev/課程選單_courses.html`

---

## ✅ 測試清單

部署完成後，依序測試：

- [ ] 開啟登入頁，輸入「不在名單」的 Email → 應該顯示錯誤
- [ ] 輸入自己的 Email（要先加入白名單） → 應該收到信
- [ ] 點擊信件連結 → 應該跳轉到課程頁
- [ ] 課程頁右上角應顯示自己的 Email
- [ ] 點任一課程 → 應該開啟遊戲
- [ ] 點「登出」→ 應該回到登入頁
- [ ] 再次開啟課程頁網址 → 因未登入，應自動跳回登入頁

## 🔧 常見問題

**Q：學員說沒收到信？**
- 請學員檢查垃圾郵件資料夾
- 在 Supabase「Authentication → Logs」查看寄信記錄

**Q：連結點了沒反應？**
- 確認 Supabase「Redirect URLs」有填正確的 `課程選單_courses.html` 完整網址

**Q：想新增更多課程？**
- 在 `courses.html` 的 `course-grid` 區塊複製貼上一個 `<a class="course-card">` 卡片，修改內容即可

**Q：免費版 Supabase 有限制嗎？**
- 免費版：**每小時僅 3 封 OTP 信件**（超過會寄送失敗！）/ 50MB 資料庫 / 最多 50,000 月活躍用戶
- ⚠️ 正式上線務必設定自訂 SMTP（建議使用 Resend，免費方案每月 3,000 封），否則多個學員同時登入會有人收不到信
- 資料庫與用戶數對你的規模完全夠用，不需要付費
